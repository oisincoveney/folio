## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Replace Flask with Reflex as the web framework. Use the Reflex CLI to scaffold the project — don't hand-roll files that the tooling generates.

Current state: Flask app lives in app.py at the project root. Frontend is Alpine.js + Jinja templates in templates/ and static/.

What to do:
- Remove `flask` from pyproject.toml dependencies; add `reflex` (latest stable). Run `uv sync`.
- Run `reflex init` at the project root. When prompted for a template, choose blank. This generates rxconfig.py, the folio/ package with __init__.py and folio.py, and the assets/ directory.
- Inspect what reflex init produced and make any necessary adjustments (e.g. confirm the app_name in rxconfig.py is "folio").
- Do NOT touch parse.py, storage.py, normalization.py, or config.py.

The existing Flask app (app.py, templates/, static/) is left in place at this stage — removed in the final subtask once the Reflex app is feature-complete.
